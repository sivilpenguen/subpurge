SubPurge App Technical Tree and Context
Generated for external review / Gemini handoff

1. Product summary

SubPurge is an Expo + React Native subscription tracking app focused on:
- tracking active and ended subscriptions
- showing renewal / expiry dates
- helping users purge unused subscriptions
- displaying accumulated spend and paid-usage estimates
- supporting dark and light theme
- supporting multiple UI languages
- scheduling local reminder notifications

Current supported locales:
- tr
- en
- es
- de
- fr

Current bottom navigation:
- Abonelikler / Subscriptions: main list and filters
- Purge Mode: review / inactivity / price-change screen
- Ayarlar / Settings: language, currency, summary, notification test, demo data


2. Runtime stack

- Expo SDK 54
- React 19.1
- React Native 0.81.5
- Expo Router
- AsyncStorage for persistence
- expo-notifications for local reminders
- @react-native-community/datetimepicker for native date pickers
- expo-image for service logos

Important config:
- app entry: expo-router/entry
- new architecture enabled
- typed routes enabled
- notifications plugin configured
- datetimepicker plugin configured


3. High-level architecture

The app is mostly organized into:
- app/: routed screens
- store/: central app state and business logic
- constants/: static app configuration, translations, themes, service presets
- utils/: date math, link opening, notifications
- components/: reusable UI pieces
- hooks/: local reusable state hooks
- assets/: bundled logos and app images
- scripts/: development / maintenance scripts
- data/: generated pricing data

The main state source of truth is:
- store/useSubscriptionStore.ts

This store handles:
- subscriptions list
- locale
- currency
- theme mode
- persistence to AsyncStorage
- derived spend math
- purge / inactivity / price-increase helpers

10. Notification system

Notification helper:
- utils/subscriptionNotifications.ts

Reminder types:
- inactivity reminder
- tracked-date reminder
- manual test notifications

Current reminder rules:
- inactivity reminder after 7 days without use
- tracked date reminder 4 days before renewal / expiry

Important behavior:
- app boot sync no longer requests notification permission automatically
- permission is requested only through an explicit user-triggered action (notification test flow)
- once permission is granted, real subscription reminders are synced again

17. Current UX flows

Add subscription flow:
1. Choose a preset or custom service
2. Select billing cycle
3. Enter or confirm price
4. Pick native start date
5. Pick native renewal or end date
6. Optionally set purpose, cancel URL, review date and notes
7. Save
8. User now receives explicit save feedback before returning

Terminate subscription flow:
1. Open edit screen
2. Tap terminate
3. Receive localized confirmation message
4. Toggle active state off

Purge flow:
1. Open Purge Mode
2. Review subscriptions due for review
3. Review idle subscriptions
4. Review price increases
5. Open manage / cancel links or update plan

Notification flow:
1. User opens Settings
2. User taps test notification button
3. App requests permission only at that moment if needed
4. Test notifications are scheduled
5. Real reminders are synced after permission is available

19. Known design characteristics and constraints

- Data is local-first and persisted on device
- No backend is currently wired for runtime pricing updates
- Pricing scrape output exists but is not yet consumed by the app
- Notifications are local notifications, not push notifications
- Many UI strings are translated, but localization quality can still be polished further
- The app is optimized around manual tracking, not external service account integrations
